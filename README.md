
# Tesla Model π Official Pre-order Platform

这是一个基于 React 19 开发的特斯拉 Model π 官方预订与品牌展示平台。本项目采用极简主义设计语言，完美契合特斯拉（Tesla）的品牌美学，专为全球 10 万部首批库存分配而设计。

## 🚀 核心功能描述

1.  **沉浸式品牌展示**：
    *   **Hero Section**：动态视觉效果，展示 Model π 的行星级设计标准。
    *   **4K 官方揭幕视频**：集成无感播放器，彻底解决生产环境下的跨域播放问题（Error 153）。
    *   **技术特性矩阵**：深度展示 Starlink（星链直连）与 Solar Charging（太阳能背板充电）技术。

2.  **高度自定义配置器**：
    *   **实时模型切换**：支持 Standard、Genesis 到 Ultra 全系列机型选择。
    *   **签名色系配置**：包括珍珠白、烈焰红、森林绿、星空紫、火焰橙、深海蓝及黑银限量版。
    *   **动态库存检测**：模拟实时库存变化，营造"首批限量"的紧迫感。

3.  **预售模式支付系统**：
    *   **分阶段付款**：用户支付商品价格的30%作为定金完成预订，剩余70%在正式开售时支付。
    *   **多渠道支持**：集成 Credit Card（信用卡）、PayPal 以及 USDT（加密货币）支付。
    *   **USDT 双协议**：支持 ERC20（以太坊）与 TRC20（波场）网络选择。
    *   **正式开售倒计时**：显示距离2026年1月31日正式开售的倒计时。

4.  **全球物流仪表盘**：
    *   **库存分配视图**：基于 Recharts 的数据可视化，展示各色系的全球配额比。
    *   **星链节点同步**：模拟 Giga Texas, Berlin, Shanghai 等全球生产节点的实时负载与物流状态。

## 🌐 SEO 优化功能

    *   **结构化数据**：Product、Organization、Website和Breadcrumb schema标记

    *   **关键词优化**：在标题、内容中合理布局核心关键词
    *   **标题结构**：优化H1-H6标题结构，提升内容层次
    *   **图片优化**：描述性ALT标签和懒加载机制

3.  **性能优化**：
    *   **资源预加载**：关键资源的预连接、预获取和预加载
    *   **图片优化**：懒加载和异步解码，提升页面速度
    *   **动态SEO**：根据页面状态动态更新meta标签

## ⚙️ 环境变量设置 (Environment Variables)

在 Vercel 等平台部署时，请在项目设置的 `Environment Variables` 中配置以下变量：

| 变量名 | 说明 | 示例值 |
| :--- | :--- | :--- |
| `API_KEY` | (可选) 若后续启用扩展功能时使用 | `your_secret_key` |
| `PAYPAL_CLIENT_ID` | 集成 PayPal 支付环境时使用 | `sb-xxxx...` |
| `USDT_WALLET_ADDR` | 接收 USDT 支付的生产钱包地址 | `Txxxxxxxx...` |

## 🚀 真实支付配置

项目已支持真实的 PayPal 和 USDT 支付功能。要启用真实支付，请：

1. **PayPal 配置**:
   - **账户类型要求**: 推荐使用 PayPal 个人账户
   - 个人账户优势: 设置简单，适合小规模交易，满足基本支付需求
   - 获取 PayPal 个人账户的 Client ID
   - 在 Vercel 环境变量中设置 `PAYPAL_CLIENT_ID`

2. **USDT 配置**:
   - 准备一个 USDT 钱包地址用于接收付款
   - 在 Vercel 环境变量中设置 `USDT_WALLET_ADDR`
   - 系统支持 ERC20 和 TRC20 网络

完成配置后，支付流程将使用真实的支付网关而不是模拟支付.

## 🛠️ 技术架构

*   **前端框架**：React 19 + TypeScript
*   **样式方案**：Tailwind CSS (响应式、动画、毛玻璃特效)
*   **图标系统**：Lucide React + Font Awesome
*   **数据可视化**：Recharts
*   **部署环境**：Vercel / Netlify (支持生产模式)

## 📅 后续开发计划 (Roadmap)

2.  **身份验证 (Authentication)**：

3.  **物流追踪系统**：

## 📌 重要说明

### PayPal 账户类型要求
- **推荐类型**: PayPal 个人账户
- **原因**: 设置简单，适合小规模交易，满足基本支付需求
- **商业账户**: 适用于高交易量场景，如需更高限额和商业功能可考虑升级

**注：应要求，本版本已移除所有 AI 交互功能，以确保生产环境的绝对稳定与快速响应。**
